# Karen - UX Design & User Flows

## Product Positioning

**One-liner:** "Your AI phone assistant that handles calls when you can't."

**Value proposition:**
- Never miss important calls
- Screen out spam and marketing
- Know who called and why - before you call back

**Target users:**
- Busy professionals
- Small business owners
- Anyone tired of spam calls

---

## Platforms

### Phase 1: Web App (MVP)
- Responsive web app (mobile-first design)
- Works on all devices via browser
- PWA capability for "app-like" experience

### Phase 2: Mobile Apps (Future)
- **Android app** (native or React Native)
- **iOS app** (native or React Native)
- Push notifications for important calls
- Quick settings toggle (enable/disable assistant)
- Deep integration with phone app (call forwarding shortcuts)

## Design Principles

1. **Mobile-first** - Design for phone screens, scale up to desktop
2. **One action per screen** - Don't overwhelm, especially in onboarding
3. **Show, don't tell** - Use example calls, not just descriptions
4. **Progressive disclosure** - Simple defaults, advanced options hidden
5. **Instant feedback** - Loading states, success confirmations
6. **Forgiveness** - Easy to undo, clear error recovery

---

## Accessibility

- Color + icon for status (not just color)
- Large tap targets (44px minimum)
- Screen reader labels for all interactive elements
- Keyboard navigation for web
- Reduced motion option
